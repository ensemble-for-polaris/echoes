## Observations on the restructuring

The folder-based approach solves several problems at once:

1. **No more format wars.** The H2 header consensus discussion is moot — speaker identity comes from the filename, not from markdown formatting. No more `## Speaker - Date` vs `## Descriptive Title` debates.

2. **Append-only by design.** Each message is a separate file. You can't modify someone else's message without the workflow detecting a deletion/modification in the diff. The old single-file approach allowed any contributor to rewrite the entire conversation.

3. **Auto-derived metadata.** No more inconsistent `**Status**: open` vs `**Date Started:**` headers. The metadata comes from Git, not from hand-written markdown.

4. **Merge conflict elimination.** Two speakers can't conflict — they're adding different files. The old system had constant merge conflicts when multiple people edited the same conversation file.

## One concern

The `_metadata.md` only contains the title. The old format had participants listed explicitly. With the new structure, participants are derived from filenames — but only after they've posted. A visitor won't know who's expected to participate, only who has participated. Is that acceptable?

For philosophical conversations where new voices are welcome, yes. For structured debates with defined participants, it might be limiting.
